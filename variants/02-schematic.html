FIG. 1.0 · LAYER36 / WORKING DRAWING

the stack, drawn end-to-end.

Source → surface → check → host. Four movements. The same .wasm leaves the developer once and lands, capability-checked, on any supported OS.

L4
HOST
adapter to OS → syscalls
Approved calls translate to native syscalls per platform. Same artifact, anywhere it lands.
TGT Win / Mac / Lin
SOON iOS / Android / Web
L3
CHECK
ucap permissions → default-deny
Every call is checked against explicit capability grants. The host decides what's allowed before anything runs.
POLICY default-deny
UNIT capability
L2
SURFACE
universal api → one dialect
A common call surface: files, processes, network, devices — spoken in one vocabulary.
SHAPE small & stable
SDK Rust (first)
L1
SOURCE
your app → .wasm
Written once, compiled to WebAssembly. No platform-specific glue. The binary is the artifact.
FORMAT WebAssembly
TOOLING cargo / wasm-tools

// NOTES, REV. A

  1. CLI accepts run, version, doctor; CI ships the same .wasm to three desktops.
  2. UCap is enforced at the runtime boundary, not inside the guest. Host knows in advance.
  3. UAPI is intentionally small. New hosts are weekends, not quarters.
  4. Threat model, benchmarks, and ADRs are in the public docs.
  5. Long-term: one app model across Win, macOS, Linux, Android, iOS, ChromeOS, and the web.
  6. Why "Layer36"? Because the right number of layers is one — but you have to draw all 36 first.
